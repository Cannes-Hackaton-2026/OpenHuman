## 2. Successes (What Went Well)
- **Technical Pivoting:** Successfully moved to a robust regex-based header validation when the `@worldcoin/agentkit` SDK proved incompatible with Next.js 16.
- **Visual Impact:** The "Agent Identity Card" provides a strong visual "Human-to-Agent" trust bridge, showing "Verified" badges for agents.
- **MCP Integration:** A fully functional MCP API allows agents to autonomously create, poll, and validate tasks.
- **Mock Mode Reliability:** Maintained a deterministic mock mode that allows testing the full identity lifecycle without external network dependencies.

## 3. Challenges (Where We Struggled)
- **SDK Compatibility:** Beta version of World AgentKit SDK caused build/runtime issues in the Next.js 16 environment.
- **Security Scoping:** Initial `validate_task` implementation lacked strict wallet-to-task ownership enforcement; caught and patched in code review.
- **Regex Duplication:** Early stories had duplicate regex logic for EVM addresses across multiple files; refactored into a central schema file.

## 4. Key Lessons & Insights
1. **SDK Fallbacks:** For hackathon environments, having a "Regex/Mock" fallback plan for unstable SDKs is critical for maintaining velocity.
2. **Identity Propagation:** In MCP architectures, authenticated identity must be explicitly propagated into tool contexts to prevent unauthorized actions.
3. **Bilateral Accountability:** The "AgentBook Link" is a unique value prop that needs to be visible in every UI interaction where an agent is involved.
